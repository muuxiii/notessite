常见模块规范 ​

CommonJS、AMD\CMD、UMD、ES Module 规范

  • CommonJS：同步 方式加载模块，nodejs 采用CommonJS规范。（在服务端，模块文件都存放在本地磁盘，读取非常快，所以这样做不会有问题。但是在浏览器端，限于网络原因，更合理的方案是使用异步模块。）
  • AMD: 异步 方式加载模块，为了在 浏览器 环境下使用，requirejs使用了AMD规范。（实现js文件的异步加载，避免网页失去响应；管理模块之间的依赖性，便于代码的编写和维护。）
  • CMD: 同AMD是采用异步加载，适用于浏览器，与AMD区别在写法。
  • UMD: UMD 糅合了 CommonJS 和 AMD 。
  • ES Module: ES模块规范，异步加载且方便模块依赖管理，可在 浏览器 和 nodejs 环境下运行。现代浏览器大多都已原生支持ES模块，对于旧版本浏览器不支持ES模块，需要用到babel编译成es5语法（是包含CommonJS规范的，编译后的js当然可以直接在node中运行，但在浏览器中运行还需要打包工具转化Commonjs语法比如browserify、webpack、rollup等）。（import...from...静态导入会被JavaScript引擎静态分析，在编译时就引入模块代码，而不是在代码运行时加载，运行时按照依赖关系执行代码，所以无法实现条件加载。也正因为这个，使得静态分析成为可能。实现条件加载可以使用import()动态导入语句异步加载）
js
// CommonJS语法

// 暴露模块: exports【暴露的是exports对象】
module.exports = value
exports.attr1 = value1

// 引入模块: require【接收的是exports对象】
var module = require('模块名/模块相对路径')

// ===============================================================

// CommonJS实践

// module1.js
// module.exports 向外暴露一个对象exports，fn1是这个对象的属性
module.exports = {
  fn1() { ... }
}
// module.exports 向外暴露一个对象exports，这个对象是个函数
// 注意: 这个exports会覆盖上面的exports。
module.exports = function () { ... }

// module2.js
// exports.xxx 向外暴露一个对象exports，xxx作为该对象的属性
exports.foo = function () {...}
exports.arr = [1,2]

// use.js
const uniq = require('uniq') // 引入第三方模块放在自定义模块上面
const module1 = require('./module1')
const module2 = require('./module2')
js
// ESM语法（分别导出比单独默认导出有利于打包优化）

// 导出模块: export
// 默认导出: 
export default 对象
// 单独导出: 
export const key1 = value1 // 导出 key1
export const obj1 = {a: 1} // 导出 对象
const a = value1
let b = value2
export {a, b} // 导出 a 、b 的语法写法，而非导出对象

// 导入模块: import
// 导入默认模块:
import xxx  from '模块路径/模块名'
// 导入单独模块
import {a, b} from '模块路径/模块名'
import * as module1 from '模块路径/模块名'

// =========================================================

// ESM实践

// module1.js
// 单独导出
export function fn1() {...}
export let fn2 = function () {...}
export const arr = [1, 2, 3]
export let str = 'str'
export let obj = { a: 1 }
const o1 = { o: 1 }
let num = 123
export { o1, num }
// 默认导出
export default {
  a: 1,
  b: 2
}

// use.js
import uniq from 'uniq' // 引入第三方模块 
import module1 from './modules/module1'  // 导入默认模块
import { fn1, fn2, arr, str, obj, o1, num } from './modules/module1' // 按需引入其他模块
import * as module from './modules/module1' // 全部引入并重命名（module的属性default为默认模块）

// ========默认导出的是值（可写js表达式），引用地址值和基本类型值；其他导出的是引用=============
// index.js
// 以下代码获得是引用
import { A } from './module.js';
import { A as otherName } from './module.js';
import * as module from './module.js';
const module = await import('./module.js');
// 以下代码获得的是值（解构赋值给了新变量）
let { A } = await import('./module.js');
// module.js
export let A = 'initial';

// 导出引用
export { A };
export { A as otherName };
// 导出值
export default A;
export default 'hello' + ' world';

commonjs和esm的主要区别 ​

  • 加载方式不同：CommonJS是同步加载模块，而ESModule是异步加载模块。
  • 导出方式不同：CommonJS 模块输出的是exports对象的引用；ES6 模块分为单独导出和默认导出，输出的是引用，是只读的，基础类型和对象类型模块输出的都是引用，对于默认导出基本类型的场景，导出的是经过js表达式计算后的值；ESM分为静态导出和动态导出，模块内代码会只加载和执行一次，后面再多次导入就不会再执行则直接复用。
  • 使用场景不同：Commonjs在nodejs环境下使用，ESM可在nodejs和浏览器环境下使用。

  1. 分析一下ES模块的执行过程？（在浏览器环境下和nodejs环境）

    • index.html

      html
      <!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Document</title>
</head>
<body>
  <script type="module" src="index.js"></script>
</body>
</html>

    • index.js

      js
      import foo from "./foo.js";
import('./dynamic.js').then(m => {
  console.log('index', m.default);
})
console.log('index',foo,bar);
import bar from "./bar.js";

    • foo.js

      js
      import bar from './bar.js'
console.log('foo',bar);
export default 'foo'

    • bar.js

      js
      console.log('bar');
export default 'bar'

    • dynamic.js

      js
      import bar from "./bar.js";
console.log('dynamic',bar);
export default 'dynamic'

    • 分析

      输出：
bar
foo bar
index foo bar
dynamic bar
index dynamic

分析：
1. 模块解析
浏览器中加载index.html后，去请求index.js文件，发现两个import，根据import的完整路径去请求foo.js和bar.js，
这两个文件下载完后，继续在foo.js中发现了import，发现bar.js已经被请求过了。
也就是会按照import的顺序去下载js文件。

2. 模块执行
先执行index.js文件，按照import的顺序去执行foo.js，在foo.js中又会去执行bar.js文件：打印 'bar'，导出 default 'bar'
回到foo.js中继续执行：打印 'foo bar'，导出 default 'foo'
回到index.js中继续执行，发现import bar.js，但不会再执行bar.js，因为这个模块已经执行过了，直接拿到import的结果 'bar' , 执行import()去请求dynamic.js，then()方法进入到微任务队列中，继续执行同步语句：打印 index foo bar
同步代码执行完后，响应完成dynamic.js，并在这个js中继续进行模块解析和模块执行，执行dynamic.js，直接拿到bar为'bar',打印 dynamic bar，导出 default 'dynamic'
再执行then中的代码，打印 index dynamic


备注：在nodejs环境下是结果是一样的，把.js改为.mjs就行，或者执行package.json中type为module。

  2. 下面代码输出结果是？

    模块是单例。无论从同一位置或不同位置导入模块多少次，模块都只会被执行和加载一次。

    js
    // index.mjs
import './module.mjs';
import { num } from './counter.mjs';
console.log('index num =', num);

// module.mjs
import { num } from './counter.mjs';
console.log('module num =', num);

// counter.mjs
if (!globalThis.num) {
  globalThis.num = 0;
}
console.log('执行了几次')
export const num = ++globalThis.num;
    shell
    # 输出
执行了几次
module num = 1
index num = 1
