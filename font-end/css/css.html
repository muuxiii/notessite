元素水平垂直居中 ​

在一个父盒子里面水平垂直居中一个子盒子（父子盒子不定宽高） ​

  • flex布局
css
.fu {
	display: flex;
	justify-content: center;
	align-items: center;
}
  • 绝对定位 + 位移
css
.fu {
	position: relative;
}
.zi {
	position: absolute;
	left: 50%;
	top: 50%;
	transform: translate(-50%,-50%);
}
  • 绝对定位 + margin: auto;
css
.fu {
	position: relative;
}
.zi {
	position: absolute;
	left: 0;
	top: 0;
	bottom: 0;
	right: 0;
	margin: auto;
}

在一个盒子中水平居中一张图片 ​

  • 上面提到的三种方式 【水平垂直居中】
  • 【行内元素行内块元素水平居中】
css
.fu {
	text-align：center;
}
  • 【块级元素水平居中】
css
.fu {
	display: block;
	margin: auto;
}

flex布局 ​

  • 给父盒子设置 display:flex; 控制子盒子的布局
  • 设置父项的属性
    • flex-direction:column; 设置主轴方向：垂直，默认row：水平
    • flex-wrap:wrap 设置是否多行显示：多行，默认nowrap：单行
    • flex-flow: [flex-direction] [flex-wrap];
    • justify-content 设置主轴上子项的排列方式
      • flex-start / flex-end / center
      • space-between 两端靠边对齐，子项间隔都相等
      • space-around 每项两侧的间隔相等。所以，项目之间的间隔比项目与边框的间隔大一倍。
      • space-evenly 平均对齐
    • align-items 设置侧轴上子项的排列方式 (单行)
      • flex-start / flex-end / center
      • stretch 伸展，未设置子项高度或者高度为auto，占满整个容器高度，默认值
    • align-content 设置侧轴上子项（每一行）的排列方式(多行)(有换行)
      • flex-start / flex-end / center
      • space-between / space-around / space-evenly / stretch
  • 设置子项的属性
    • flex: [flex-grow] [flex-shrink] [flex-basis];
      • flex属性是flex-grow, flex-shrink和 flex-basis的简写，默认 0 1 auto。flex: 1 即 flex: 1 1 auto。有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。
      • flex-grow：定义项目的的放大比例，存在剩余空间的情况下
        • 默认0，即使存在剩余空间，也不会放大
        • 例如有三个子项：第一个1，第二个2，第三个0，那么第一个放大到占剩余空间的1/3，第二个2/3，第三个不放大
      • flex-shrink：定义项目的缩小比例，剩余空间不足的情况下
        • 默认为1，即如果空间不足，该项目将缩小（所有项目都为1：当空间不足时，缩小的比例相同）
        • 为0：即如果空间不足，该项目不会缩小
      • flex-basis：定义在分配多余空间之前，项目占据的主轴空间，浏览器根据此属性计算主轴是否有多余空间
        • 默认值为auto，即项目原本大小
        • 设置后项目将占据固定空间
    • align-self允许单个项目在侧轴上有不一样的排列方式，可覆盖 align-items
    • order 排序

position定位 ​

给了fixed或position定位，脱离文档流，若不给宽高，默认是内容的宽高，如果也没有内容，则宽高都是0；浮动也是如此。

  • static 静态定位，即没有定位，用于取消定位
  • fixed 固定定位，相对于窗口，脱离文档流（不占位置）
  • relative 相对定位，相对于自身，不脱离文档流（占位置，原来自身的位置任然在文档流中占据，新位置也占据）
  • absolute 绝对定位，相对于第一个有定位的父元素，脱离文档流（不占位置）
  • absolute和relative的区别
    • relative只有两个位置值（如果left、right、top、bottom都存在就只保留left和top）
    • absolute可以写4个位置值都生效（left: 0;right:0;top:0;bottom:0）

清除浮动 ​

浮动元素会脱离文档流：父元素不设置高度，如果子元素不设置浮动，则父元素的高度会由子元素撑开；如果子元素浮动了，则父元素的高度就会变成 0 （由于子元素浮动造成父元素高度塌陷的现象）；

清除浮动就是为了解决这个问题。

清除浮动的方式 ​

html
<style>
#a{
  width: 300px;
  background-color: pink;
  /* overflow:hidden; 父元素添加overflow:hidden;触发bfc  */ 
}
/* 给父元素添加after伪元素，设置clear:both */
#a:after { content:''; display:block; clear:both;}

#b{
  float: left;
  height: 50px;
  width: 50px;
  background-color: rgb(48, 132, 181);
}
#c{
  float: left;
  height: 100px;
  width: 100px;
  background-color: rgb(45, 182, 111);
}
</style>

<div id="a">
    <div id="b"></div>
    <div id="c"></div>
</div>

总结 ​

scss
// minin.scss （因为考虑到复用，写到scss的混入中）
// 清除浮动
@mixin clearfix {
  &:after {
    content: "";
    display: table;
    clear: both;
  }
}
scss
// 在scss中使用

    @include clearfix;
}

BFC (Formatting Contexts) ​

  • BFC是块级格式化上下文，指一个独立的块级渲染区域，内部元素如何排列，与外部区域无关，不影响外部，与外部隔离起来了。
  • 如何触发BFC
    • overflow:hidden; 不为 visible
    • 设置了float浮动，（脱离文档流）
    • position是绝对定位或固定定位，（脱离文档流的不占位置）
    • display是inline-block flex
  • BFC的实际作用
    • 解决外边距塌陷(外边距合并)可以给父盒子设置overflow:hidden;,触发bfc规则。
    • 浮动造成的高度塌陷

隐藏元素 ​

  • 隐藏方式：
    • display:none; 不占空间
    • visibility: hidden; 占空间
    • opacity: 0; 占空间
    • 使用定位移出可视区; 用不占空间的定位
  • display:none 与 visibility: hidden 区别
    • display:none 不占空间 visibility: hidden 占空间
    • display:none 引起回流和重绘 visibility: hidden 引起重绘
  • overflow:hidden; 这只是溢出隐藏，注意区分

消除图片底边距 ​

  • 原因：行内元素默认vertical-align属性是基线（baseline）对齐，父元素的底边框对应着行内元素的底线（bottom）对齐，所以默认情况下行内元素会与父元素产生底部间隙。
  • 解决
    • 设置image为块级元素
    • vertical-align:bottom; （只要不是baseline）

浏览器渲染引擎渲染过程 ​

浏览器会将HTML解析成DOM树，将CSS解析成CSSOM树，将DOM和CSSOM结合成Render渲染树。根据渲染树计算元素尺寸位置，然后绘制页面像素信息，最后由GPU显示到屏幕上

重绘和回流(重排) ​

  • 重绘：元素的颜色、背景、边框样式（不涉及到布局的、位置尺寸的）改变，浏览器会将新的样式赋予元素并重新绘制

  • 回流：渲染树中的一些元素的尺寸、位置、布局改变时，浏览器会重新渲染部分或全部DOM,回流一定引起重绘。

  • 减少重绘与回流：

    • 避免频繁操作DOM，采用文档片段，采用虚拟DOM
    • 减少不必要的DOM层级
    • 尽量给DOM树的叶节点添加样式，如直接给img标签设置宽高
    • 为产生动画的元素使用 fixed absolute
    • display:none 引起回流和重绘 visibility:hidden 引起重绘

  • 为什么有时候人们用translate来改变位置而不是定位？ translate()是transform的一个值。transform会使用GPU硬件加速，性能更好，不会触发重绘回流；position + top/left会触发大量的重绘和回流，性能影响较大。

css选择器及优先级 ​

  • 优先级
    • !important 加在某个属性值后面 （优先级最高）
    • 行内样式 （权重 1,0,0,0）
    • id选择器 （权重 0,1,0,0）
    • 类选择器、属性选择器、伪类选择器 （权重 0,0,1,0）
    • 元素选择器、伪元素选择器 （权重 0,0,0,1）
    • 继承 或 * （权重 0,0,0,0）
  • 选择器

    • 基础选择器

      • 元素选择器、类选择器、id选择、通配符选择器*

    • 复合选择器

      • 并集选择器（分组的意思，或的关系）,
      • 交集选择器（既是又是的关系，与的关系）没空格紧挨着

    • 关系选择器（元素关系：父元素、子元素、祖先元素、后代元素、兄弟元素）

      • 后代选择器 （选后代元素，空格）
      • 子选择器 (选子元素)>
      • 下一个兄弟元素 + 例如 div > p选择div的下一个兄弟（紧挨着的）元素且是p元素
      • 下面所有兄弟元素~例如div > p选择div下面的所有兄弟元素且是p元素

    • 属性选择器 []

      • [属性名] 选择具有某属性的元素
      • [属性名=属性值] 选择属性值等于某属性值的元素
      • [属性名^=属性值] 选择属性值以某属性值开头的元素
      • [属性名$=属性值] 选择属性值以某属性值结尾的元素
      • [属性名*=属性值] 选择属性值含有某属性值的元素

    • 伪类选择器 : (伪类用于描述元素的特殊状态：如第一个子元素、被点击的元素、鼠标移入的元素...)

      • 子元素位置（第一类）

        • :first-child 第一个子元素（所有子元素中的），如 ul>li:first-child 选择第一个子元素且是li
        • :last-child
        • :nth-child(3) 第3个
        • :nth-child(n) 第n个（n>=0），:nth-child(0) 就第0个，代表没有
        • :nth-child(2n) or :nth-child(evev) 第偶数个
        • :nth-child(2n+1) or :nth-child(odd) 第奇数个

      • 子元素位置（第二类：在同类型元素中进行排序，与第一类不同按照所有元素排序）

        • :first-of-type
        • :last-of-type
        • :nth-of-type()

      • 排除

        • :not(选择器) 如 li:not(:nth-of-type(2)) 选择所有li除了第二个

      • 链接伪类 （有 LVHA 顺序）

        • a:link 未访问过的（正常的链接）
        • a:visited 已访问过的（但是只能用于修改链接的颜色）
        • :hover鼠标移入的状态【所有元素也适用】
        • :active鼠标点击不松开的状态【所有元素也适用】

      • 结构伪类

        • input:focus 用于选取获得焦点的表单元素

    • 伪元素选择器 ::

      • ::first-letter 选中首字符 ::first-line 选中首行
      • ::selection 鼠标选中的内容
      • ::before 选择首元素（必须有content属性，content的内容无法通过鼠标选中）
      • ::after 选择尾元素（必须有content属性，content的内容无法通过鼠标选中）

有哪些方式（CSS）可以隐藏页面元素？ ​

  • opacity:0 元素的透明度设为0，占文档流，可以交互
  • visibility:hidden 占文档流，不可交互
  • display:none 删除了元素，不占文档流，不可交互
  • overflow:hidden 这个只隐藏元素溢出的部分，占文档流，不可交互

em\px\rem区别？ ​

  • px：绝对单位，页面按精确像素展示。
  • em：相对单位，基准点为父节点字体的大小，如果自身定义了font-size按自身来计算，1em = 1(font-size)px。 (浏览器默认字体大小是16px)。
  • rem：相对单位，可理解为”root em”, 相对根节点html的字体大小来计算。
