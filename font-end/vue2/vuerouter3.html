Vue Router 3.x ​

SPA单页应用的路由插件：vue2 适用 Vue Router 3.x ；vue3 适用 Vue Router 4.x

题外话：前后端路由 ​

  • 前端路由：路径对应的是组件，地址栏路径改变时，展示对应的组件
  • 后端路由：路径对应是的函数。根据服务器收到的请求的路径匹配对应的函数，处理客户端请求，返回响应数据

路由模式：hash、history、abstract ​

hash（默认） ​

特点：地址带#号，#及后面内容（路径和参数）就是hash值，http请求的时候hash值不会作为路径的一部分带给服务器，兼容性好

原理：监听window.onHashChange事件，hash变化后（hash值变化，页面不会刷新；a链接可以改变hash 或者 window.hash = "#/path"）， 触发事件处理函数，获取hash值，去路由表中匹配，将匹配到的组件放到指定容器中渲染。

history ​

特点：地址不带#号；刷新页面出现404，因为刷新页面会向服务器发送get请求，请求路径就是路由 /path，所以需要服务器配置处理（如nginx:try_files $uri $uri/ /admin/index.html;）

原理：H5的History API。history对象有个历史记录池，记录每次切换的地址，基于History API切换路由页面不会刷新。切换路由后，获取最新的路由地址，去路由表中匹配，匹配到的组件在容器中渲染；也可以基于window.onpopstate事件监听，会在back/forward/go 方法后进行触发。

  • history.pushState 跳转到新的地址，记录池中添加该地址（最新的一条记录）
  • history.replaceState 跳转到新的地址，记录池中替换当前地址
  • history.back / history.forward / history.go

abstract（非浏览器环境中路由方式，比如 Node、weex 等） ​

支持所有javascript运行模式。如果发现没有浏览器的API，路由会自动强制进入这个模式。没有使用浏览器api，可以放到node环境或者桌面应用中。

基本原理 ​

  • 路由表 + 路由容器 + 路由切换（编程式导航 / 组件导航）
  • 扩展能力
    • 导航路由守卫
    • 路由懒加载
    • ...

Vue.use(VueRouter) 调用install方法，全局混入一个beforeCreate和一个destroyed钩子，

js
// install 源码
Vue.mixin({
  beforeCreate () {
    if (isDef(this.$options.router)) { // 当前vue实例的选项上有router对象，一般就是在根实例上传入router对象
      this._routerRoot = this; // （根实例上）添加属性 _routerRoot 为当前vue实例
      this._router = this.$options.router; // （根实例上）添加属性 _router 为 router 对象
      this._router.init(this); // 调用VueRouter原型上的init方法
      Vue.util.defineReactive(this, '_route', this._router.history.current); // 对 _route 做响应式劫持
    } else {
      // 一般非root组件上，走到这里，都会拿到root根实例上的 _routerRoot 挂载到一般组件上的 _routerRoot
      this._routerRoot = (this.$parent && this.$parent._routerRoot) || this;
    }
    registerInstance(this, this);
  },
  destroyed () {
    registerInstance(this);
  }
});
// 向vue原型上定义 $router  即VueRouter实例
Object.defineProperty(Vue.prototype, '$router', {
  get () { return this._routerRoot._router }
});
// 向vue原型上定义 $route  即路由表信息（在当前组件上获取是最新的路由信息）
Object.defineProperty(Vue.prototype, '$route', {
  get () { return this._routerRoot._route }
});

// 注册两个全局组件 RouterView 路由容器 RouterLink 路由切换
Vue.component('RouterView', View);
Vue.component('RouterLink', Link);

  • history 路由模式： http://xxx/user/007?a=1&b=2#abc

  query:{a: '1', b: '2'}, // URL查询参数，如果没有就是 {}
  redirectedFrom: '',   // 如果存在重定向，即为重定向来源的路由的名字。如果不是，就没有这个属性
}

路由跳转的方式 ​

  • 页面第一次加载时, localhost:8080 ，会根据路由模式，自动的在地址后面加一些内容（如 hash => localhost:8080/#/ ； history => localhost:8080/ ），此时也路由跳转了一次
    • 如果路由表中有匹配，会根据路由表中匹配的进行跳转 [{path: '/', redirect: '/home'}, {path: '/home', component: Home}]，于是 path: '/' => path: '/coms'
    • 如果路由表中没有匹配，于是 path: '/' => path: '/'
  • 路由跳转有两种方式
    1. 声明式导航
      • <router-link to=""> 同 push()
      • <router-link :to="" replace> 同 replace()
    2. 编程式导航
      • $router.push()
      • $router.replace()
      • $router.go() 跳转到某条历史记录，但不会新增一条历史记录
      • $router.forward() 前进 同 router.go(1)
      • $router.back() 后退 同 router.go(-1)

路由跳转的传参方案 ​

从A组件（销毁）跳到B组件（渲染）

除了路由跳转传参，完全可以通过“组件通信”方案（如vuex），实现两个组件间的信息共享/传递

问号传参（查询/query参数） ​

传参的信息显示在地址栏中，以问号的形式展示 ?a=1&b=2

跳转后，手动刷新页面，也可以获取查询参数，因为query信息在地址栏中存在，每次路由匹配解析都可以获取到

js
this.$router.push('/b?a=1&b=2') // 方式1：path (带 ? 的)
this.$router.push({ path: '/b', query: { plan: 'private' }}) // 方式2： path + query
router.push({ name: 'b', query: { plan: 'private' }}) // 方式3：name + query
js
this.$route.query // 获取查询参数

隐式传参（params参数） ​

只能通过命名路由跳转，传递的信息不展示在地址栏

跳转后，手动刷新页面，传递的信息会丢失

js
router.push({ name: 'b', params: { id: '123' }})  // 方式1：name + query
js
this.$route.params // 获取params参数

路径传参（path参数） ​

只能通过路径跳转，传递的信息展示在地址栏，以路径的形式展示（和query一样，区别就是在地址栏展示的形式）

跳转后，手动刷新页面，也可以获取参数，因为参数信息在地址栏中存在

js
{
  path: '/b/:p1?/:p2?', // ? 可选 ，没有传时，值为undefined
  name: 'b',
  component: () => import('@/views/routes/B.vue')
}
js
router.push({ path: '/b/1/2' })  // 方式1：path (带 / 的)
js
this.$route.params // 获取path参数  {p1: "1", p2: "2"}
// $route
// {
//   fullPath: "/b/1/2",
//   params: {p1: '1', p2: '2'}
//   path: "/b/1/2"
// }

// 当手动改变地址栏路径的时候，比如 /b/1/2 => /b/1/5
// 手动刷新页面后，获取的是 {p1: "1", p2: "5"}
js
// 当使用一个通配符时，$route.params 内会自动添加一个名为 pathMatch 参数。它包含了 URL 通过通配符被匹配的部分
// 给出一个路由 { path: '/user-*' }

this.$route.params.pathMatch // 'admin'
// 给出一个路由 { path: '*' }

this.$route.params.pathMatch // '/non-existing'

props传参 ​

将组件与路由解耦，为了组件的灵活性（一般是可复用的组件）

背景

javascript
// User 组件
const User = {
  // 组件与路由耦合了：组件内部使用了 $route （组件只能在特定的 URL 上使用，限制了组件的灵活性）
  template: '<div>User {{ $route.params.id }}</div>'   
}
// router
const router = new VueRouter({
  routes: [{ path: '/user/:id', component: User }]
})

做法

js
// 通过 props 解耦：路由上定义props，传给组件接受props
const User = {
  props: ['id'],
  template: '<div>User {{ id }}</div>'    // 组件与路由解耦了
}

// router
const router = new VueRouter({
  routes: [
    { path: '/user/:id', component: User, props: true },

    // 解耦了就可以用于其他路由了（这样你便可以在任何地方使用该组件，使得该组件更易于重用和测试）
    {path: '/somepath', component: User }
  ]
})
细节

布尔模式

如果 props 被设置为 true，route.params 将会被设置为组件属性。把路由收到的所有【params对象参数】通过props传给组件，所以在组件中可以通过props接收。【来源于此时route的params】

对象模式

如果 props 是一个对象，它会被按原样设置为组件属性。当 props 是静态的时候有用。该对象中所有的key-value的组合最终都会通过props传给组件，所以在子组件中可以通过props接收，但都是死数据，所以用的少。【在props中定义的不会跑到当前route上的params上】

js
const router = new VueRouter({
  routes: [
    {
      path: '/promotion/from-newsletter',
      component: Promotion,
      props: { newsletterPopup: false }
    }
  ]
})

const Promotion= {
  props: ['newsletterPopup'],
  template: '<div>{{ newsletterPopup }}</div>' 
}

函数模式

创建一个函数返回 props。这样你便可以将参数转换成另一种类型，将静态值与基于路由的值结合等等。props只会在路由发生变化时起作用

js
const router = new VueRouter({
  routes: [
    {
      path: '/search',
      component: SearchUser,
      props: route => ({ query: route.query.q })  // URL /search?q=vue 会将 {query: 'vue'} 作为属性传递给 SearchUser 组件
	  // props($route){
          // return {
              //id: $route.query.id,
              // title: $route.query.title
          // }
      // }
    }
  ]
})

const SearchUser= {
  props: ['query'],
  template: '<div>{{ query}}</div>' 
}

// 总结
//第一种写法：对象，该对象中所有的key-value的组合最终都会通过props传给messageDetail组件，所以在子组件中可以通过props接收，但都是死数据，所以用的少。
//props:{ a:1,b:2 },

</script>

路由懒加载 ​

如果不用路由懒加载，打包后"主js"文件太大了，导致第一次页面加载时间长（请求资源的时间+渲染的时间）。

  • 第一次需要渲染的组件打包到"主js"中，防止白屏可做骨架屏处理；
  • 其余的每个路由组件或者每几个路由组件打包到不同的js中，每当切换路由后，再请求资源加载（后面就被浏览器缓存了），即路由懒加载。
js
const Main = () => import('./Main.vue')
const router = new VueRouter({
  routes: [
    // 打包后：js/app.[hash].js  css/app.[hash].css
    { path: '/main', component: Main }, // 不使用路由懒加载
    // 打包后：js/714.[hash].js  css/714.[hash].css
    { path: '/foo', component: () => import('.//Foo.vue') } // 使用路由懒加载
    // 打包后：js/group-foo.[hash].js   css/group-foo.[hash].css
    { path: '/bar', component: () => import(/* webpackChunkName: "group-foo" */ './Bar.vue') }, // 使用路由懒加载
    { path: '/baz', component: () => import(/* webpackChunkName: "group-foo" */ './Baz.vue') }, // 使用路由懒加载
  ]
})

路由守卫 ​

完整导航流程

  1. 导航被触发。在失活的组件里调用 beforeRouteLeave 【组件内守卫】。
  2. 调用 beforeEach ，路由切换开始 【全局前置守卫】。
  3. 【特殊场景A组件跳A组件】在重用的组件里调用 beforeRouteUpdate 【组件内守卫】。
  4. 进入路由表中进行匹配，调用 beforeEnter【路由守卫】。
  5. 在被激活的组件里调用 beforeRouteEnter【组件内守卫】。
  6. 调用 beforeResolve【全局解析守卫】。导航被确认。
  7. 调用 afterEach 路由切换完毕【全局后置守卫】。
  8. 触发 DOM 更新。再触发组件内生命周期钩子。

从 A 组件跳转到 B 组件：

  • A beforeRouteLeave
  • beforeEach
  • beforeEnter
  • B beforeRouteEnter
  • beforeResolve
  • afterEach
  • B beforeCreate
  • B created
  • B beforeMount
  • A beforeDestroy
  • A destoryed
  • B mounted

从 A 组件跳转到 A 组件：

  • beforeEach
  • A beforeRouteUpdate
  • beforeResolve
  • afterEach
js
//全局前置守卫
// to 和 from 是两个route ，to 是目的路由，from 是源路由，next()是放行和跳转
router.beforeEach((to,from,next)=>{
   if(to.meta.isAuth){ //判断要去的路由是否进行权限控制，需要配置meta
   	if(localStorage.getItem('school') === 'atguigu'){ //权限控制的具体规则
   		next() //放行
   	}else{
   		next({name:'qita'}) // 跳转路由
   	}
   }else{
   	  next() //放行
   }
})

// next()
// 一定要调用该方法来 resolve 这个钩子。执行效果依赖 next 方法的调用参数
// next(): 进行管道中的下一个钩子。如果全部钩子执行完了，则导航的状态就是 confirmed (确认的)。
// next(false): 中断当前的导航。如果浏览器的 URL 改变了 (可能是用户手动或者浏览器后退按钮)，那么 URL 地址会重置到 from 路由对应的地址。
// next('/') 或者 next({ path: '/' }): 跳转到一个不同的地址。当前的导航被中断，然后进行一个新的导航。需要设置条件进行 next(path)，否则会进入死循环。你可以向 next 传递任意位置对象，且允许设置诸如 replace: true、name: 'home' 之类的选项以及任何用在 router-link 的 to prop 或 router.push 中的选项。
// next(error): (2.4.0+) 如果传入 next 的参数是一个 Error 实例，则导航会被终止且该错误会被传递给 router.onError() 注册过的回调。

// 全局解析守卫 beforeResolve
// 在 2.5.0+ 你可以用 router.beforeResolve 注册一个全局守卫。这和 router.beforeEach 类似，区别是在导航被确认之前，同时在所有组件内守卫和异步路由组件被解析之后，解析守卫就被调用。

//全局后置守卫
// 没有next，切换已经完成了
router.afterEach((to,from)=>{
   document.title = to.meta.title || 'vue_test_my'
})


// 路由独享守卫：`beforeEnter` 
new VueRouter({
  routes: [
    {
      path: '/foo',
      component: Foo,
      beforeEnter: (to, from, next) => {
        // ...
      }
    }
  ]
})

// 组件内守卫
const Foo = {
    template: `...`,
    // beforeRouteEnter 守卫 不能 访问 this，因为守卫在导航确认前被调用，因此即将登场的新组件还没被创建。
    // 不过，你可以通过传一个回调给 next来访问组件实例。在导航被确认的时候执行回调，并且把组件实例作为回调方法的参数。
    beforeRouteEnter(to, from, next) {
        // 进入守卫：通过路由规则，进入该组件时被调用
        next(vm => {
            // 通过 `vm` 访问组件实例
        })
    },
    // 注意 beforeRouteEnter 是支持给 next 传递回调的唯一守卫。对于 beforeRouteUpdate 和 beforeRouteLeave 来说，this 已经可用了，所以不支持传递回调，因为没有必要了。
    beforeRouteUpdate(to, from, next) {
        // 可以用来监听动态路由参数变化
        // 在当前路由改变，但是该组件被复用时调用
        // 举例来说，对于一个带有动态参数的路径 /foo/:id，在 /foo/1 和 /foo/2 之间跳转的时候，
        // 由于会渲染同样的 Foo 组件，因此组件实例会被复用。而这个钩子就会在这个情况下被调用。
        // 可以访问组件实例 `this`
        this.name = to.params.name
        next()
    },
    // 这个离开守卫通常用来禁止用户在还未保存修改前突然离开。该导航可以通过 next(false) 来取消。
    beforeRouteLeave(to, from, next) {
        // 离开守卫：通过路由规则，离开该组件时被调用 
        // 导航离开该组件的对应路由时调用
        // 可以访问组件实例 `this`
        const answer = window.confirm('Do you really want to leave? you have unsaved changes!')
        if (answer) {
            next()
        } else {
            next(false)
        }
    }
}

缓存路由组件 ​

让不展示的路由组件保持挂载，不被销毁。

html
<!--News是组件名-->
<keep-alive :include="['News','Message']"> 
   <router-view></router-view>
</keep-alive>

<!--也可以这么做-->
<keep-alive>
  <router-view v-if="$route.meta.keepAlive"/>
</keep-alive>
<router-view v-if="!$route.meta.keepAlive"/>

  • deactivated 路由组件失活（消失了）时触发
